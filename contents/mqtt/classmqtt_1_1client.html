inline

Set a keep alive second and a pimg milli seconds.

Parameters
keep_alive_seckeep alive seconds
ping_msping sending interval

When a client connects to a broker, the client notifies keep_alive_sec to the broker. After connecting, the broker starts counting a timeout, and the client starts sending ping packets for each ping_ms. When the broker receives a ping packet, timeout timer is reset. If the broker doesn't receive a ping packet within keep_alive_sec, the client is disconnected.
See http://docs.oasis-open.org/mqtt/mqtt/v3.1.1/os/mqtt-v3.1.1-os.html#_Toc398718030
